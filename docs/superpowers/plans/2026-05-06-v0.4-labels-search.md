**Status:** planned (decisions locked 2026-05-07)
**Predecessor:** v0.3 (projects + issues CRUD), tag `v0.3.0`, HEAD `40cf9e1`
**Theme:** label CRUD with a fixed color palette, multi-facet filter + search + pagination on the issue list, label assignment on create/edit/detail.

## Decisions (locked via Q&A 2026-05-07)

| # | Decision | Choice |
|---|---|---|
| 1 | Color storage | Postgres native enum `label_color` |
| 2 | Color palette (12) | `gray, red, orange, yellow, green, teal, blue, indigo, violet, pink, brown, slate` |
| 3 | Label edit | name + color both editable via PATCH |

| 13 | Label assignment API | per-label POST / DELETE |
| 14 | Label embedding in IssueDto | embedded `labels: [{id, name, color}]` |
| 15 | Label delete with assignments | confirm dialog showing count, then hard delete + cascade |
| 16 | Multi-value URL param shape | comma-separated (`?status=todo,in_progress`) |

### V13 — `label_color` enum

```sql
create type label_color as enum (
  'gray','red','orange','yellow','green','teal',
  'blue','indigo','violet','pink','brown','slate'
);
```

### V14 — `label` table

```sql
create table label (
  id          uuid                     not null default uuidv7() primary key,
  created_at  timestamp with time zone not null default current_timestamp,
  created_by  varchar(64)              not null,
  updated_at  timestamp with time zone not null default current_timestamp,
  updated_by  varchar(64)              not null,
  version     integer                  not null default 0,
  project_id  uuid                     not null references project(id) on delete cascade,
  name        varchar(50)              not null,
  color       label_color              not null
);
```

### V15 — unique label name per project (case-insensitive)

```sql
create unique index uq_label_project_id_name on label (project_id, lower(name));
```

(Matches the email-uniqueness pattern in v0.2.)

### V16 — set_updated_at trigger on label

```sql
create trigger label_set_updated_at
before update on label
for each row execute function set_updated_at();
```

### V17 — `issue_label` join table

```sql
create table issue_label (
  id          uuid                     not null default uuidv7() primary key,
  created_at  timestamp with time zone not null default current_timestamp,
  created_by  varchar(64)              not null,
  updated_at  timestamp with time zone not null default current_timestamp,
  updated_by  varchar(64)              not null,
  version     integer                  not null default 0,
  issue_id    uuid                     not null references issue(id) on delete cascade,
  label_id    uuid                     not null references label(id) on delete cascade,
  unique (issue_id, label_id)
);
```

### V18 — set_updated_at trigger on issue_label

```sql
create trigger issue_label_set_updated_at
before update on issue_label
for each row execute function set_updated_at();
```

### V19 — generated tsvector on issue

```sql
alter table issue
add column search_tsv tsvector
generated always as (
  to_tsvector('english',
    coalesce(title, '') || ' ' || coalesce(description_md, '')
  )
) stored;
```

### V20 — GIN index for FTS

```sql
create index ix_issue_search_tsv on issue using gin (search_tsv);
```

## REST API

### Labels (project-scoped)

| Method | Path | Body | Returns | Notes |
|---|---|---|---|---|
| GET | `/api/projects/{KEY}/labels` | — | `LabelDto[]` | sorted by `name asc` |
| POST | `/api/projects/{KEY}/labels` | `{name, color}` | `LabelDto` (201) | 409 on duplicate name (case-insensitive) |
| GET | `/api/projects/{KEY}/labels/{id}` | — | `LabelDto` | 404 if not in project |
| PATCH | `/api/projects/{KEY}/labels/{id}` | `{name?, color?}` | `LabelDto` | non-null fields only; 409 on rename collision |
| DELETE | `/api/projects/{KEY}/labels/{id}` | — | 204 | cascades `issue_label` rows |

### Issue ↔ label assignment

| Method | Path | Body | Returns | Notes |
|---|---|---|---|---|
| POST | `/api/projects/{KEY}/issues/{N}/labels` | `{labelId}` | `IssueDto` (200) | idempotent — already-assigned returns 200; 400 if label not in project |
| DELETE | `/api/projects/{KEY}/issues/{N}/labels/{labelId}` | — | 204 | idempotent — unknown assignment returns 204 |

### Issue list (extended)

`GET /api/projects/{KEY}/issues` now accepts:

| Param | Shape | Default | Max | Behavior |
|---|---|---|---|---|
| `status` | comma-separated enum values | none | — | OR within facet |
| `priority` | comma-separated enum values | none | — | OR within facet |
| `label` | comma-separated UUIDs | none | — | OR within facet (issue has any of these labels) |
| `q` | string | none | — | `websearch_to_tsquery('english', q)` against `search_tsv` |
| `page` | int | 0 | — | zero-indexed |
| `size` | int | 50 | 100 | clamped server-side |

Response:

```json
{
  "items": [IssueDto, ...],
  "total": 142,
  "page": 0,
  "size": 50
}
```

Across-facet logic is AND. Order is `number desc`. No sort param in v0.4.

### Issue create (extended)

`POST /api/projects/{KEY}/issues` body now accepts `labelIds`:

```json
{
  "title": "...",
  "description": "...",
  "status": "todo",
  "priority": "medium",
  "dueDate": null,
  "labelIds": ["uuid", "uuid"]
}
```

Service validates each `labelId` exists and belongs to this project, then bulk-inserts into `issue_label` inside the same transaction as the issue insert.

## Acceptance criteria per major behavior

1. **Create a label and use it on an issue** — `POST /labels` then `POST /issues` with `labelIds` returns the issue with `labels[0]` populated; the label survives a refresh of the SPA.
2. **Filter combinator** — issue with status=`todo` + label=`bug` matches `?status=todo,in_progress&label=<bug-id>,<api-id>` because `todo` ∈ {todo, in_progress} AND label ∈ {bug, api}.
3. **FTS** — issue with description containing "auth middleware breaks on logout" matches `?q=auth+logout` via `websearch_to_tsquery`.
4. **Cascade** — deleting a label removes its `issue_label` rows; deleting a project removes its labels and join rows; `mvn verify` E2E proves both.
5. **URL state** — refresh of a filtered/paginated/searched view restores the same view; back button works; URL is shareable.
6. **Label-not-in-project** — assigning a label from project A to an issue in project B returns 400 with a clear error.
