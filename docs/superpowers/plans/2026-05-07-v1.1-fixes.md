## Backend

### New endpoint
`GET /api/setup-required` → `{"required": boolean}`. CSRF-exempt, no auth. Returns `true` when the `account` table is empty, `false` otherwise. Used by the SPA's auth gate.

Add to `SetupController`:
```java
@GetMapping("/api/setup-required")
public Map<String, Boolean> setupRequired() {
  return Map.of("required", setupService.isRequired());
}
```
`SetupService.isRequired()` = `accountDataService.count() == 0`.

Spring Security: `requestMatchers("/api/setup-required").permitAll()`.

## Acceptance criteria per major behavior

1. **Login flow round-trip** — fresh user opens any deep link → redirected to /login → logs in → lands on the original URL. AuthGate handles the probing on app mount; intendedRoute survives a full reload.
2. **Setup → Login transition** — after the very first setup completes, the AuthGate's next mount returns `required: false` from `/api/setup-required` and routes any logged-out user to /login.
3. **Activity completeness** — every label assignment emits `label_added` (whether via /labels POST OR via issue create with labelIds OR via IssueForm edit). Every label removal emits `label_removed` (whether via /labels DELETE on an assigned label OR via the unassign endpoint OR via cascade from label delete).
4. **Markdown anchor purity** — every `<a>` produced by `<Markdown>` has only the expected attributes (`href`, `target`, `rel` where applicable). No `node`, no nested anchors. Single visit-per-pixel of focusable area.
5. **Locale consistency** — every validation error message AND every formatted number on the page renders in English/US convention. No Hibernate Validator German leak. No `10.000` separator surprise.
6. **API hygiene** — POST/create responses include `createdAt`/`updatedAt`; ProjectDto excludes `nextIssueNumber`; all 400 responses share the same ProblemDetail shape.
